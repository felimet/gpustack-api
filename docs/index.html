開始使用 › 概述

GPUStack 部署手冊

單節點 LLM / VLM / Embedding / Rerank 推論服務，透過 Cloudflare Tunnel 對外暴露 API。Server / Worker 分離，Docker Compose 一鍵啟動，主機不需開放入站埠。

整體架構

外部用戶端（curl / OpenAI SDK / 瀏覽器）
       │  HTTPS
       ▼
Cloudflare Edge ─────────────────────────────────────────────
  ├─ sub-api.yourdomain.com/v1  →  CF Access Service Token（程式化 API）
  └─ sub.yourdomain.com      →  CF Access Email 白名單（Web UI）
       │
       ▼  outbound-only Tunnel（主機不開放入站埠）
cloudflared 容器（172.100.10.4）
       │
       ▼  http://gpustack:80
gpustack server 容器（172.100.10.2）
  Web UI  ·  Higress gateway  ·  API
       │
       ▼  depends_on: service_healthy
gpustack-worker 容器（172.100.10.3）
  NVIDIA Runtime  ·  docker.sock
       │
       ▼  GPUStack 動態產生推論容器
llama.cpp / vLLM 推論容器（--network host，0.0.0.0:PORT）
  路由：Higress → 172.100.10.1:PORT → host bridge gateway
       │
       ▼
NVIDIA GPU 硬體（RTX 3080 × 4）

網路設計

容器固定 IP角色
gpustack 172.100.10.2Server — Web UI + Higress gateway（無 GPU）
gpustack-worker 172.100.10.3Worker — NVIDIA Runtime + 推論容器管理
gpustack-cloudflared172.100.10.4Cloudflare Tunnel 出口節點
bridge gateway（宿主機） 172.100.10.1推論路由出口；host network 介面

網路 / 存取 › Cloudflare Tunnel

Cloudflare Tunnel

主機只需 outbound 443，無需開放任何入站埠。

雙 Hostname 策略

HostnameCF Access用途
sub-api.yourdomain.comService Token程式化 API
sub.yourdomain.com Email 白名單Web UI 管理員

建立 Tunnel

Zero Trust → Networks → Tunnels → Create a tunnel（Cloudflared）。複製 Token 填入 .env。

Public Hostnames

SubdomainService URLAdditional Settings → Access
sub-api.yourdomain.comhttp://gpustack:80不填（留空）
sub.yourdomain.com http://gpustack:80不填
🚨
Additional Settings → Access 必須留空。填了 Application 後，cloudflared 啟用 originRequest.access.required:true，Service Token JWT（"type":"service_token"）被 QUIC 層靜默丟棄，CF edge 回 502，無任何 HTTP 日誌。

502 診斷

# API 請求在 log 中完全沒有 HTTP request log
docker compose logs cloudflared --tail=50
docker compose logs cloudflared --tail=50

修正：CF Dashboard → Public Hostnames → 編輯 sub-api → Additional settings → Access 清空 → Save。等 30 秒 cloudflared 拉取新設定。

網路 / 存取 › Service Token

Service Token

程式化 API 的 Cloudflare 身份憑證，獨立於 GPUStack API Key。

建立

Zero Trust → Access → Service Auth → Service Tokens → Create Service Token

  • Name：gpustack-api-client
  • Duration：1 年或 No expiration
⚠
Secret 只顯示一次，立即保存 CF-Access-Client-Id 與 CF-Access-Client-Secret。

模型 › VLM mmproj 設定

VLM mmproj 設定

GPUStack v2.1.2 部署 GGUF VLM 時，mmproj 不自動傳入 llama.cpp。

⚠
症狀："image input is not supported"。根因：GPUStack 下載 mmproj 但不自動傳 --mmproj 給 llama.cpp。

mmproj 快取路徑（container 內）

/var/lib/gpustack/cache/huggingface/<ORG>/<REPO>/mmproj-F32.gguf

API 方式

# 1. 查詢 model id
$models = Invoke-RestMethod -Uri "http://localhost:8089/v2/models" `
  -Headers @{ Authorization = "Basic $([Convert]::ToBase64String([Text.Encoding]::ASCII.GetBytes('admin:<PASSWORD>')))" }
$models.items | Select-Object id, name

# 2. 更新 backend_parameters
$body = @{
  source            = "local_path"
  local_path        = "<GGUF 路徑>"
  name              = "<MODEL_NAME>"
  backend           = "llama.cpp"
  backend_parameters = @("--mmproj <mmproj 完整路徑>")
  replicas          = 1
} | ConvertTo-Json
Invoke-RestMethod -Method PUT `
  -Uri "http://localhost:8089/v2/models/<MODEL_ID>" `
  -Headers @{ Authorization = "Basic ..." } `
  -ContentType "application/json" -Body $body

# 3. 刪除 instance，強制以新參數重啟
Invoke-RestMethod -Method DELETE `
  -Uri "http://localhost:8089/v2/model-instances/<INSTANCE_ID>" `
  -Headers @{ Authorization = "Basic ..." }
# 1. 查詢 model id
curl -s http://localhost:8089/v2/models \
  -u "admin:<PASSWORD>" \
  | python3 -c "import sys,json;[print(x['id'],x['name']) for x in json.load(sys.stdin)['items']]"

# 2. 更新 backend_parameters
curl -s -X PUT "http://localhost:8089/v2/models/<MODEL_ID>" \
  -u "admin:<PASSWORD>" \
  -H "Content-Type: application/json" \
  -d '{
    "source":"local_path","local_path":"<GGUF 路徑>",
    "name":"<MODEL_NAME>","backend":"llama.cpp",
    "backend_parameters":["--mmproj <mmproj 完整路徑>"],
    "replicas":1
  }'

# 3. 刪除 instance
curl -s -X DELETE "http://localhost:8089/v2/model-instances/<INSTANCE_ID>" \
  -u "admin:<PASSWORD>"

進階參考 › Entrypoint 腳本

Entrypoint 腳本

WSL2 環境 workaround。腳本在容器內（Linux）執行，以下僅顯示容器內的 Bash 內容。

entrypoint-server.sh

bash#!/bin/sh
set -e
DATA=/var/lib/gpustack
mkdir -p "$DATA"

# WSL2 無 /sys/class/dmi/id/product_uuid → 讀 /proc/sys/kernel/random/uuid
if [ ! -f "$DATA/worker_uuid" ]; then
    UUID=$(cat /proc/sys/kernel/random/uuid 2>/dev/null || \
           python3 -c "import uuid; print(uuid.uuid4())")
    printf '%s' "$UUID" > "$DATA/worker_uuid"
fi

# 預寫 token 檔案（覆蓋 GPUStack 自動生成行為）
if [ -n "$GPUSTACK_TOKEN" ] && [ ! -f "$DATA/token" ]; then
    printf '%s' "$GPUSTACK_TOKEN" > "$DATA/token"
fi

exec /usr/bin/entrypoint.sh "$@"

entrypoint-worker.sh

bash#!/bin/sh
set -e
DATA=/var/lib/gpustack
mkdir -p "$DATA"

if [ ! -f "$DATA/worker_uuid" ]; then
    UUID=$(cat /proc/sys/kernel/random/uuid 2>/dev/null || \
           python3 -c "import uuid; print(uuid.uuid4())")
    printf '%s' "$UUID" > "$DATA/worker_uuid"
fi

# WSL2 NVML 回傳 float，Python 無法對 float 做 << 位移 → int() 轉型
RUNTIME_PY=$(python3 -c \
    "import gpustack.detectors.runtime.runtime as m; print(m.__file__)" 2>/dev/null || true)
if [ -n "$RUNTIME_PY" ] && grep -q "dev\.memory << 20" "$RUNTIME_PY" 2>/dev/null; then
    sed -i \
        's/dev\.memory << 20/int(dev.memory) << 20/g;
         s/dev\.memory_used << 20/int(dev.memory_used) << 20/g' \
        "$RUNTIME_PY"
fi

exec /usr/bin/entrypoint.sh "$@"

問題根因修正

呼叫 API

GPUStack 相容 OpenAI API 格式（/v1/ 前綴）。本機直連或透過 Cloudflare Tunnel 皆可使用 OpenAI SDK / curl 呼叫。

Base URL 與認證

情境Base URL需要的 Header
本機直連 http://localhost:8089/v1 Authorization: Bearer <api-key>
Cloudflare Tunnel（外網） https://sub-api.yourdomain.com/v1 CF-Access-Client-Id
CF-Access-Client-Secret
Authorization: Bearer <api-key>
ℹ
GPUStack API Key 在 Web UI → Settings → API Keys 建立。此 Key 僅用於 Authorization header，與 CF Service Token 獨立管理。

GET /v1/models — 列出已部署模型

# 本機
Invoke-RestMethod -Uri "http://localhost:8089/v1/models" `
  -Headers @{ Authorization = "Bearer <api-key>" } |
  Select-Object -ExpandProperty data | Select-Object id, object

# 外網（Cloudflare Tunnel）
Invoke-RestMethod -Uri "https://sub-api.yourdomain.com/v1/models" `
  -Headers @{
    "CF-Access-Client-Id"     = "<client-id>"
    "CF-Access-Client-Secret" = "<client-secret>"
    "Authorization"           = "Bearer <api-key>"
  } | Select-Object -ExpandProperty data | Select-Object id
# 本機
curl -s http://localhost:8089/v1/models \
  -H "Authorization: Bearer <api-key>" \
  | jq '.data[].id'

# 外網（Cloudflare Tunnel）
curl -s https://sub-api.yourdomain.com/v1/models \
  -H "CF-Access-Client-Id: <client-id>" \
  -H "CF-Access-Client-Secret: <client-secret>" \
  -H "Authorization: Bearer <api-key>" \
  | jq '.data[].id'

POST /v1/chat/completions — 文字對話

$body = @{
  model    = "<model-name>"
  messages = @(
    @{ role = "system";  content = "You are a helpful assistant." }
    @{ role = "user";    content = "你好，請介紹一下你自己。" }
  )
  max_tokens   = 512
  temperature  = 0.7
} | ConvertTo-Json -Depth 5

Invoke-RestMethod -Method POST `
  -Uri "http://localhost:8089/v1/chat/completions" `
  -Headers @{ Authorization = "Bearer <api-key>" } `
  -ContentType "application/json" `
  -Body $body |
  Select-Object -ExpandProperty choices |
  Select-Object -First 1 |
  ForEach-Object { $_.message.content }
curl -s http://localhost:8089/v1/chat/completions \
  -H "Authorization: Bearer <api-key>" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "<model-name>",
    "messages": [
      {"role": "system",  "content": "You are a helpful assistant."},
      {"role": "user",    "content": "你好，請介紹一下你自己。"}
    ],
    "max_tokens": 512,
    "temperature": 0.7
  }' | jq '.choices[0].message.content'

Python SDK Streaming

pythonfrom openai import OpenAI

client = OpenAI(
    base_url="http://localhost:8089/v1",
    api_key="<api-key>",
)

stream = client.chat.completions.create(
    model="<model-name>",
    messages=[{"role": "user", "content": "用繁體中文解釋量子糾纏"}],
    stream=True,
)
for chunk in stream:
    delta = chunk.choices[0].delta.content or ""
    print(delta, end="", flush=True)
print()  # 換行

POST /v1/embeddings — 向量嵌入

$body = @{
  model = "<embedding-model-name>"
  input = @("這是第一段文字", "這是第二段文字")
} | ConvertTo-Json

Invoke-RestMethod -Method POST `
  -Uri "http://localhost:8089/v1/embeddings" `
  -Headers @{ Authorization = "Bearer <api-key>" } `
  -ContentType "application/json" `
  -Body $body |
  Select-Object -ExpandProperty data |
  ForEach-Object { "index $($_.index): dim=$($_.embedding.Count)" }
curl -s http://localhost:8089/v1/embeddings \

  -H "Content-Type: application/json" \

    "model": "<embedding-model-name>",
    "input": ["這是第一段文字", "這是第二段文字"]
  }' | jq '.data[] | {index, dim: (.embedding | length)}'

常見錯誤代碼

HTTP 狀態碼原因解決方式
401 Unauthorized API Key 錯誤或遺漏 確認 Authorization: Bearer <key> header 正確
403 Forbidden CF Access 驗證失敗 確認 CF-Access-Client-Id/Secret 正確；Token 未過期
404 Not Found 模型名稱不存在 / 無 running instance用 GET /v1/models 確認可用模型 ID
422 Unprocessable 請求 JSON 欄位錯誤 確認 model、messages 欄位格式正確
502 Bad Gateway CF Tunnel Access 設定錯誤 見 CF Tunnel → 502 診斷
503 Service Unavailable模型正在載入中 等待推論容器啟動（Web UI → Models 確認狀態）
